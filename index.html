Introduction

Hector is a location transparent web-framework written in Scala which makes heavy use of Akka.

Routing

Requests are routed with a partial function. It could not be simpler than that.

val userProfileActor = context.actorOf(Props[UserProfileActor])

def routes = {
  case Get("user" /: publicKey /: _) ⇒  Route(userProfileActor, Some(publicKey → true))
}

Hector is always specific. A requests ends with a Required_/ or No_/ but in this case we do not care so we use _.

HTML

Hector comes with an excellent HTML emitter that is more secure and faster than Scala's built-in XML Utility.

It can generate beautiful and human readable HTML code with proper indentation or omit all unnecessary whitespace. But it will never break your content like Utility.trim does.

JavaScript

If HTML is salt, JavaScript is pepper and both together create a spicy combination. Hector mixes pepper with steroids so that you get awesome JavaScript support.

Hector comes with typed top-level definitions which evaluate to an AST. window.alert(window.status.length * 2) is actual Scala code which is compiled and minified on the server.

You can either write JavaScript using the typed top-level and implicit conversions or write it completely manual. Here are two examples that both result in (function(){hector.execCallback('...')})() on the client.

// Generate the AST manually:

JsCall(
  callee = JsFunc(
    id = None,
    parameters = Seq.empty,
    body = JsBlock(
      JsCall(
        JsMember(JsIdentifier('hector), JsIdentifier('execCallback)),
        Seq(callbackName)
      )
    )
  ),
  arguments = Seq.empty
)

// Using implicits and the DSL (will result in same AST):

(function { ('hector ~> 'execCallback)(callbackName) })()
